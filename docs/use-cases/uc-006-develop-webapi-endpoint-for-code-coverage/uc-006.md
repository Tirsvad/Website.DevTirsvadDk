## User story
As a stakeholder, I want to retrieve code coverage information through the API so that I can assess the current automated test coverage of the project.

### Brief Use Case
**Title**: Develop WebAPI endpoint for code coverage  
**Success Flow**: The team provides a WebAPI endpoint that returns the latest available coverage report summary in a format the frontend can display clearly.

### Fully Dressed Use Case
**Title**: Develop WebAPI endpoint for code coverage  
**Scope**: dev.tirsvad.dk project status API  
**Level**: User goal  
#### Actors:  
- Stakeholder  
- Web Platform  
- Developer  
#### Related Requirements:  
- REQ-F-004  
#### Preconditions:  
- Coverage reports can be produced or their absence can be detected.  
- The API foundation is available.  
#### Main Flow:  
1. The developer defines the route, response model, and freshness rules for coverage data.  
2. The developer implements backend logic that locates and reads the latest approved coverage report.  
3. The Web Platform requests the coverage endpoint.  
4. The endpoint validates the request and transforms the coverage data into the response model.  
5. The endpoint returns the coverage summary for frontend display.  
#### Extensions:

&nbsp;&nbsp;&nbsp;&nbsp;a: At any time, the report format changes.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The developer updates the parser or mapping logic.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The endpoint is revalidated with the new format.  

&nbsp;&nbsp;&nbsp;&nbsp;2a: No report is available.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The endpoint returns a controlled response that states coverage data is unavailable.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The absence is logged for follow-up.  
&nbsp;&nbsp;&nbsp;&nbsp;4a: The report is stale or incomplete.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The endpoint includes status information that the data is not current.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The caller can still display the last known result with the warning.  
#### Postconditions:  
- Consumers can retrieve current or last known code coverage information.  
- Missing or stale reports are surfaced clearly.
