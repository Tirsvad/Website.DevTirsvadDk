## User story
As a project team member, I want to conduct user testing and gather stakeholder feedback so that we can confirm the platform works for its intended audience.

### Brief Use Case
**Title**: Conduct user testing and gather feedback  
**Success Flow**: The team runs planned user testing sessions, captures observations and feedback, and records the findings for follow-up improvements.

### Fully Dressed Use Case
**Title**: Conduct user testing and gather feedback  
**Scope**: dev.tirsvad.dk validation process  
**Level**: User goal  
#### Actors:  
- Test Participant  
- Project Team  
- Web Platform  
#### Related Requirements:  
- REQ-U-001  
#### Preconditions:  
- A testable version of the platform is available.  
- The team has a user testing plan and representative participants.  
#### Main Flow:  
1. The Project Team prepares the test objectives, tasks, and success measures.  
2. The Test Participant performs the defined tasks on the platform.  
3. The Project Team observes the participant and records notable findings.  
4. The Project Team gathers direct verbal or written feedback after the tasks.  
5. The Project Team consolidates the results into an actionable feedback set.  
#### Extensions:

&nbsp;&nbsp;&nbsp;&nbsp;a: At any time, a planned task no longer reflects the current product.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The Project Team updates the task definition.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The revised task is used for the remaining sessions.  

&nbsp;&nbsp;&nbsp;&nbsp;1a: The plan is missing coverage for a key workflow.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The team expands the test plan before running the session.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The session starts only when the gap is addressed.  
&nbsp;&nbsp;&nbsp;&nbsp;3a: The participant encounters a blocking usability issue.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The issue is documented with context and evidence.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The team decides whether to continue, adapt, or stop the session.  
#### Postconditions:  
- The team has structured feedback from user testing.  
- Findings are available to drive refinement decisions.
