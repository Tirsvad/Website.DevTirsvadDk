## User story
As a stakeholder, I want blog posts about project status so that I can follow progress updates and interact with the team through comments and likes.

### Brief Use Case
**Title**: Implement blog post feature  
**Success Flow**: The team implements a blog feature that supports publishing status posts and enables users to comment on and like those posts.

### Fully Dressed Use Case
**Title**: Implement blog post feature  
**Scope**: dev.tirsvad.dk blog capability  
**Level**: User goal  
#### Actors:  
- Stakeholder  
- Content Author  
- Web Platform  
#### Related Requirements:  
- REQ-F-005  
#### Preconditions:  
- The frontend and backend foundation are available.  
- Blog content rules and interaction expectations are defined.  
#### Main Flow:  
1. The Content Author creates or edits a project status post.  
2. The Web Platform validates the post content against the publishing rules.  
3. The Web Platform publishes the blog post for stakeholders to read.  
4. The Stakeholder opens the published post.  
5. The Stakeholder submits a comment or a like through the approved interaction flow.  
#### Extensions:

&nbsp;&nbsp;&nbsp;&nbsp;a: At any time, interaction rules change.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The team updates the blog interaction logic and validation rules.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The revised feature is retested.  

&nbsp;&nbsp;&nbsp;&nbsp;2a: The post fails validation.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The platform prevents publication.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The Content Author corrects the post and retries.  
&nbsp;&nbsp;&nbsp;&nbsp;5a: The stakeholder action is invalid, duplicated, or blocked by policy.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The platform rejects the action with a controlled response.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The stakeholder remains on the post page with a clear message.  
#### Postconditions:  
- The platform supports publishing and viewing project status blog posts.  
- Stakeholders can interact with blog posts through comments and likes when allowed.
