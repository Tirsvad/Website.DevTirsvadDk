## User story
As a user, I want an authentication API endpoint so that I can sign in securely from the web application.

### Brief Use Case
**Title**: Develop WebAPI endpoint for authentication  
**Success Flow**: The team exposes an authentication endpoint that accepts approved credentials, validates identity, and returns the result needed by the client application.

### Fully Dressed Use Case
**Title**: Develop WebAPI endpoint for authentication  
**Scope**: dev.tirsvad.dk access API  
**Level**: User goal  
#### Actors:  
- User  
- Web Platform  
- Developer  
#### Related Requirements:  
- REQ-F-003  
- REQ-SEC-001  
#### Preconditions:  
- Authentication rules and identity infrastructure are available.  
- The client application knows the approved sign-in flow.  
#### Main Flow:  
1. The developer defines the endpoint route, request model, response model, and security controls for sign-in.  
2. The user submits credentials through the approved client experience.  
3. The endpoint validates the request payload and forwards the credentials to the authentication service.  
4. The authentication service verifies the credentials and determines the sign-in result.  
5. The endpoint returns a secure success or failure response to the client application.  
#### Extensions:

&nbsp;&nbsp;&nbsp;&nbsp;a: At any time, the authentication policy changes.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The developer updates the endpoint contract or validation behavior.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The endpoint is retested before release.  

&nbsp;&nbsp;&nbsp;&nbsp;3a: Request validation fails.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The endpoint rejects the request with a clear validation result.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: No authentication attempt is performed.  
&nbsp;&nbsp;&nbsp;&nbsp;4a: The credentials are invalid or the account is blocked.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: The endpoint returns a failure result without granting access.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: The event is logged according to security policy.  
#### Postconditions:  
- Approved users can attempt sign-in through a secure API contract.  
- Failed authentication attempts are handled consistently.
