# Glossary for dev.tirsvad.dk Project

| Term | Definition |
|------|------------|
| Risk | The possibility of an event or condition that may negatively impact project objectives. |
| Mitigation | Actions taken to reduce the likelihood or impact of a risk. |
| Stakeholder | Any individual or group affected by or with an interest in the project. |
| Data refresh | The process of updating project data to ensure accuracy and timeliness. |
| Validation routines | Procedures to check the correctness and completeness of project data. |
| Audit | A systematic review to ensure compliance and data integrity. |
| Authentication | The process of verifying the identity of a user or system. |
| Authorization | The process of granting or denying access to resources. |
| Encryption | The process of encoding data to prevent unauthorized access. |
| Regression testing | Testing to confirm that new changes do not adversely affect existing functionality. |
| Caching | Storing frequently accessed data for faster retrieval. |
| Disaster recovery | Strategies and processes for restoring data and services after a failure. |
| GDPR | General Data Protection Regulation, a European privacy law. |
| Continuous integration (CI) | The practice of automatically building and testing code changes. |
| CI/CD pipeline | An automated workflow that validates, packages, and delivers changes through continuous integration and continuous delivery stages. |
| Code review | The process of examining code for quality and correctness. |
| Blazor | A .NET web UI framework used to build interactive frontend components. |
| Blog post | A published project update or article that stakeholders can read and interact with. |
| Doxygen | A documentation generator used to produce code documentation from source code and comments. |
| Gateway | A milestone checkpoint that groups related delivery objectives and exit criteria. |
| Milestone | A planned delivery checkpoint used to measure project progress. |
| Project status | The current summary of project progress, risks, quality, and supporting artifacts. |
| Quality criteria | The documented measures and expectations used to assess the quality of the solution. |
| README | A primary project document that introduces the solution, setup, and usage information. |
| Supply chain risk | Risks arising from dependencies on external vendors or components. |
| User testing | Structured evaluation sessions in which representative users perform tasks and provide feedback. |
| UX | User experience; the overall usability and clarity of interacting with the platform. |
| WebAPI | An HTTP-based API used by the platform to expose backend functionality and project data. |
| Ownership | Responsibility for maintaining and updating project information. |
| Feedback channel | Mechanism for stakeholders to provide input or report issues. |
| CrossReference | See also: [risk-and-mitigation.md](risk-and-mitigation.md) |
